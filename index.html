octopress中国行

由于众所周知的原因，octopress的安装和使用可能和官方说明有些不同的地方，本文记录了在github上使用octopress搭建博客的过程。

请注意以下
所有操作均在windows7中完成

安装并配置git

这里有详细说明

创建一个仓库，命名为
username.github.com

其中username为你在github的用户名
注意只有以这个名字命名该仓库，github才能自动将其识别为你的主页
否则你建立的博客将不能通过username.github.com访问到。

下载安装ruby

点击这里下载安装ruby193

添加环境变量RUBY_HOME为你的ruby安装目录（如D:\Ruby193）
在环境变量PATH后追加";%RUBY_HOME%\bin"(引号不用输入)

安装并配置DevKit

点击这里下载DevKit-tdm-32-4.5.2
 如果你选择使用其他版本的ruby,
请到官网下载对应版本的DevKit

从github上clone octopress到本地

git clone git://github.com/imathis/octopress.git octopress
cd octopress
ruby --version # Should report Ruby 1.9.3

修改gem和bundle的源

由于众所周知的原因，rubygems.org无法访问，
因此需要把gem和bundle的源改为国内的镜像。
修改gem源
$gem sources --remove "https://rubygems.org/"
$ gem sources -a "http://ruby.taobao.org/"
$ gem sources -l
*** CURRENT SOURCES ***
http://ruby.taobao.org
# 请确保只有 ruby.taobao.org
修改octopress项目bundle源
将上一步clone下来的octopress中Gemfile文件的source改为http://ruby.taobao.org
source "http://ruby.taobao.org"

安装依赖项

gem install bundler
bundle install
安装Octopress的默认主题
rake install

配置环境变量 LANG 和 LC_ALL

配置这两个环境变量的目的是避免
下一步出现因字符编码而导致的错误
请在Git Bash中敲入以下命令：
set $LANG=zh_CN.UTF-8
set $LC_ALL=zh_CN.UTF-8
echo "export LANG LC_ALL">~/.bash_profile

rake deploy

提示输入 要写你实际的仓库名 分清最后是.io 还是.com
git tag 1.0.0 1b2e1d63ff
1b2e1d63ff 是你想要标记的提交 ID 的前 10 位字符。使用如下命令获取提交 ID：
git log
你也可以用该提交 ID 的少一些的前几位，只要它是唯一的。

替换本地改动

假如你做错事（自然，这是不可能的），你可以使用如下命令替换掉本地改动：
git checkout -- <filename>
此命令会使用 HEAD 中的最新内容替换掉你的工作目录中的文件。已添加到缓存区的改动，以及新文件，都不受影响。

假如你想要丢弃你所有的本地改动与提交，可以到服务器上获取最新的版本并将你本地主分支指向到它：
git fetch origin
git reset --hard origin/master

有用的贴士

内建的图形化 git：
gitk
彩色的 git 输出：
git config color.ui true
显示历史记录时，只显示一行注释信息：
git config format.pretty oneline
交互地添加文件至缓存区：
git add -i
